# Practical 02 Assignment: PostgreSQL Performance Analysis and Optimization

## Objective

Learn how to analyze PostgreSQL performance under load, identify inefficient SQL queries, locking issues, problems caused by wide tables, missing indexes, and propose technical solutions to improve database performance.

## Assignment

You are provided with a Python script **load_assignment_generator.py** that:

1. Creates a PostgreSQL database.
2. Creates tables for a sample CRM/e-commerce system.
3. Populates the tables with test data.
4. Executes a set of SQL queries that generate database load.
5. Simulates common performance issues, including:

    * slow queries;
    * locking and blocking;
    * inefficient JOIN operations;
    * queries on non-indexed columns;
    * updates on a wide table;
    * aggregations over large volumes of data.

## Student Tasks

Analyze the database behavior while the script is running.

You are expected to independently identify:

1. Which queries generate the highest load on the database.
2. Where locking or blocking occurs.
3. Which tables or columns require indexing.
4. Which queries should be rewritten or optimized.
5. Which tables should be normalized or split into smaller structures.
6. Which PostgreSQL mechanisms can be applied to improve performance.

## Expected Activities

Students should use PostgreSQL tools and techniques such as:

* `EXPLAIN`
* `EXPLAIN ANALYZE`
* `pg_stat_activity`
* `pg_locks`
* `pg_stat_statements`
* indexes
* partial indexes
* composite indexes
* JOIN optimization
* table normalization
* partitioning (where appropriate)
* `VACUUM` / `ANALYZE`
* transaction logic optimization

## Deliverables

Prepare a short report describing:

1. The issues that were identified.
2. How each issue was detected.
3. Which SQL queries were problematic.
4. What changes were implemented.
5. The results achieved after optimization.
6. A comparison of performance before and after the changes.

## Constraints

There is no single correct solution. Students are expected to independently select appropriate optimization techniques and justify their decisions.

## Tips

Query history:

```sql
CREATE EXTENSION pg_stat_statements; -- If not exitsts in current DB

SELECT
    query,
    calls,
    total_exec_time,
    mean_exec_time,
    rows
FROM pg_stat_statements
ORDER BY total_exec_time DESC
LIMIT 100;
```

-- Identify blocked sessions and their corresponding blocking sessions
```sql
SELECT
blocked.pid AS blocked_pid,
blocked.usename AS blocked_user,
blocked.query AS blocked_query,
blocking.pid AS blocking_pid,
blocking.usename AS blocking_user,
blocking.query AS blocking_query,
blocked.wait_event_type,
blocked.wait_event
FROM pg_stat_activity blocked
JOIN pg_stat_activity blocking
ON blocking.pid = ANY(pg_blocking_pids(blocked.pid))
ORDER BY blocked.pid;
```

-- Detect long-running transactions that may be causing lock contention
```sql
SELECT
pid,
usename,
state,
xact_start,
now() - xact_start AS transaction_duration,
query
FROM pg_stat_activity
WHERE xact_start IS NOT NULL
ORDER BY xact_start;
```

-- List active sessions currently waiting to acquire a lock
```sql
SELECT
pid,
usename,
state,
wait_event_type,
wait_event,
query_start,
now() - query_start AS running_for,
query
FROM pg_stat_activity
WHERE wait_event_type = 'Lock';
```
